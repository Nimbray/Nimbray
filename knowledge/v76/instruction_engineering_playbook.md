# V76 — Instruction Engineering Playbook

Une bonne instruction IA doit :

- définir l'identité sans surjouer ;
- préciser la mission ;
- préciser le ton ;
- indiquer les limites ;
- donner des règles de réponse ;
- prévoir les cas sensibles ;
- interdire l'invention de liens, sources, fichiers ou capacités ;
- demander de respecter les contraintes utilisateur ;
- forcer l'IA à reconnaître l'incertitude.

Mauvaise instruction : "Sois la meilleure IA du monde."

Meilleure instruction : "Réponds de façon utile, fiable, structurée et humaine. Identifie la demande principale, respecte les contraintes, signale l'incertitude, refuse les demandes dangereuses et ne prétends jamais avoir créé un fichier ou accédé à une source sans preuve."
