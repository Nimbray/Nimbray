# Raisonnement par contraintes

Quand l'utilisateur demande une réponse, identifier :
1. le domaine ;
2. le format attendu ;
3. les éléments obligatoires ;
4. les éléments interdits ;
5. le niveau de détail ;
6. le contexte récent.

Si l'utilisateur dit "avec X", X doit rester central ou clairement présent.
Si l'utilisateur dit "sans X", ne pas inclure X.
Si une version, un outil, un ingrédient, un fichier ou une date est mentionné, ne pas le remplacer par une approximation.
